# 全文审查检查清单

## 一、情节逻辑检查项

### 1.1 前后矛盾检查
- [ ] 人物行为是否与前文设定矛盾
- [ ] 事件结果是否与前文描述矛盾
- [ ] 时间线是否一致（日期、时间间隔）
- [ ] 地点是否一致（人物位置是否合理）
- [ ] 物品/道具是否凭空出现或消失
- [ ] 人物能力/状态是否突然变化

### 1.2 逻辑漏洞检查
- [ ] 情节发展是否有足够的铺垫
- [ ] 关键转折是否有合理的原因
- [ ] 人物做出选择是否有足够动机
- [ ] 事件发生是否符合世界观规则
- [ ] 解决方案是否过于巧合或突兀

### 1.3 因果链检查
- [ ] 每个事件是否有明确的原因
- [ ] 每个事件是否有合理的结果
- [ ] 事件之间是否有清晰的因果关系
- [ ] 是否有"无因之果"或"无果之因"

### 1.4 动机检查
- [ ] 主角行为是否有清晰的动机
- [ ] 配角行为是否有合理的动机
- [ ] 反派行为是否有逻辑自洽的动机
- [ ] 人物动机是否与性格一致

### 1.5 合理性检查
- [ ] 世界观规则是否自洽
- [ ] 力量体系是否平衡
- [ ] 社会结构是否合理
- [ ] 人物成长速度是否合理

---

## 二、人物一致性检查项

### 2.1 性格一致性
- [ ] 人物性格是否前后一致
- [ ] 人物是否有OOC（Out of Character）行为
- [ ] 人物的价值观是否稳定
- [ ] 人物的习惯/癖好是否一致

### 2.2 行为一致性
- [ ] 人物行为是否符合其性格设定
- [ ] 人物决策方式是否一致
- [ ] 人物面对相似情况的反应是否一致
- [ ] 人物的能力水平是否稳定

### 2.3 成长弧线
- [ ] 人物是否有合理的成长变化
- [ ] 成长是否有足够的铺垫
- [ ] 成长是否符合人物性格
- [ ] 成长是否与情节发展同步

### 2.4 关系一致性
- [ ] 人物关系是否稳定
- [ ] 关系变化是否有合理的原因
- [ ] 人物之间的互动是否符合关系设定
- [ ] 关系描述是否前后一致

### 2.5 称呼一致性
- [ ] 同一人物的称呼是否统一
- [ ] 人物之间的称呼是否符合关系
- [ ] 不同场景下的称呼变化是否合理

---

## 三、叙事视角检查项

### 3.1 视角一致性
- [ ] 是否始终保持同一视角
- [ ] 视角切换是否清晰明确
- [ ] 是否有视角混乱的段落

### 3.2 视角越界检查
- [ ] 是否展示了非视角角色的内心想法
- [ ] 是否描述了视角角色无法看到的场景
- [ ] 是否揭示了视角角色不知道的信息

### 3.3 视角切换检查
- [ ] 视角切换是否有明确的标识
- [ ] 切换是否过于频繁
- [ ] 切换是否造成读者困惑

---

## 四、伏笔回收检查项

### 4.1 伏笔识别
- [ ] 是否有明显的伏笔埋设
- [ ] 伏笔是否被正确识别和追踪
- [ ] 是否有隐藏的伏笔未被发现

### 4.2 伏笔回收
- [ ] 已埋设的伏笔是否全部回收
- [ ] 回收是否自然，不突兀
- [ ] 回收是否与前文呼应
- [ ] 回收是否有足够的铺垫

### 4.3 逻辑闭环
- [ ] 主线是否闭环
- [ ] 副线是否闭环
- [ ] 是否有未解决的悬案
- [ ] 结局是否完整

---

## 五、节奏把控检查项

### 5.1 节奏分布
- [ ] 快慢节奏是否合理分布
- [ ] 是否有连续过快或过慢的段落
- [ ] 节奏变化是否自然

### 5.2 拖沓检查
- [ ] 是否有可以删减的冗余段落
- [ ] 描述是否过于冗长
- [ ] 是否有重复表达
- [ ] 是否有不必要的细节描写

### 5.3 仓促检查
- [ ] 是否有发展过快的情节
- [ ] 关键情节是否缺少铺垫
- [ ] 情感变化是否过于突然
- [ ] 事件过渡是否自然

### 5.4 爽点间隔
- [ ] 爽点/高潮之间的间隔是否合理
- [ ] 爽点强度是否有层次感
- [ ] 是否有长时间无爽点的段落

---

## 六、文风统一检查项

### 6.1 用词统一
- [ ] 专业术语是否统一
- [ ] 人物称呼是否统一
- [ ] 地名/组织名是否统一
- [ ] 设定术语是否统一

### 6.2 句式风格
- [ ] 句式风格是否一致
- [ ] 句子长度是否有变化
- [ ] 是否有明显的风格断裂

### 6.3 对话风格
- [ ] 不同角色的对话是否有辨识度
- [ ] 对话是否符合人物身份
- [ ] 对话是否自然，不生硬

### 6.4 AI味检测
- [ ] 是否有过度工整的排比句
- [ ] 是否有"眼中闪过一丝..."等AI高频表达
- [ ] 是否有过于书面化的描述
- [ ] 是否有机械的对话标签（"他说道"）

---

## 七、语病与用词检查项

### 7.1 语法错误
- [ ] 是否有明显的语法错误
- [ ] 句子成分是否完整
- [ ] 标点使用是否规范

### 7.2 用词不当
- [ ] 是否有用词不准确的情况
- [ ] 是否有搭配不当的情况
- [ ] 是否有用词重复的情况

### 7.3 重复冗余
- [ ] 是否有语义重复的表达
- [ ] 是否有可以合并的段落
- [ ] 是否有不必要的修饰词

### 7.4 标点问题
- [ ] 标点使用是否符合中文规范
- [ ] 是否有错用标点的情况
- [ ] 是否有遗漏标点的情况

---

## 八、审查优先级

### P0：必须修改（严重问题）
- 情节逻辑严重矛盾
- 人物OOC行为
- 视角严重混乱
- 伏笔完全遗漏
- 明显的语法错误

### P1：建议修改（中等问题）
- 情节逻辑不够严谨
- 人物性格不够稳定
- 节奏不够合理
- 文风不够统一
- 用词不够准确

### P2：可选修改（轻微问题）
- 微小的用词优化
- 句式调整
- 标点规范
- 风格微调

---

## 九、修改原则

1. **最小修改原则**：能改一个词就不改一句，能删一句就不重写一段
2. **保留原意原则**：修改只改"怎么写"，不改"写什么"
3. **一致性原则**：修改后必须与整体保持一致
4. **可读性原则**：修改后的文本必须易于阅读
5. **风格统一原则**：修改必须符合整体文风
